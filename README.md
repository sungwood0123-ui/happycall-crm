# 세찬컴퍼니 인트라넷 V29.52

## V29.52 변경사항

- 모바일 직원별 해피콜 현황을 긴 표 대신 핵심 정보 카드 목록으로 변경
- 모바일 통화 불가 고객 목록이 보이지 않던 문제를 수정하고 카드 목록·페이지 이동 적용
- 모바일 메뉴에 누락된 해피콜 배정 현황 메뉴 추가
- 모바일 배정 현황의 직원 목록과 대상 목록을 위아래로 배치하여 화면 눌림·한 글자 줄바꿈 제거
- 모바일 감사로그에서 PC 표와 모바일 카드가 중복되지 않도록 표시 구조 정리
- 직원별 현황 조회 중 공통 로딩 화면을 표시하여 빈 화면 오해 방지
- PC 상단 메뉴가 1280px 화면에서 오른쪽으로 잘리지 않도록 폭과 글자 크기 자동 조정
- 기존 해피콜·프리패스 생성 및 저장 규칙은 변경하지 않음

## V29.52 QA 결과

- 자동 테스트 9건 통과
- 운영 데이터 기준 로그인 및 주요 화면 조회 확인
- PC 1280px 상단 메뉴·직원별 현황 표·오류보고 표 정상 표시 확인
- 모바일 390px 직원별 현황·통화 불가 고객·감사로그·오류보고 카드 표시 확인
- 모바일 배정 현황 메뉴 진입 및 직원 목록·대상 목록 세로 배치 확인
- 오류보고 상세 팝업의 닫기 버튼·내부 스크롤·글자 잘림 없음 확인
- 운영 배포 전 사용자 최종 승인 대기

## V29.51 변경사항

- 전체 해피콜·검수·배정 현황·직원별 현황에서 화면에 필요한 데이터만 조회하도록 최적화
- 매장·직원별 화면은 운영 DB에서 필요한 범위만 조회하여 불필요한 전체 다운로드 제거
- 통화 불가 고객·전체 해피콜·검수·배정 현황·프리패스 로그에 100건 단위 페이지 이동 적용
- 통화 불가 고객의 관련 고객·처리 이력 조회를 안전한 동시 처리 방식으로 개선하고 로딩 상태 표시
- 감사로그·오류보고는 최신 500건을 DB에서 바로 정렬하여 조회
- 프리패스 로그에서 목록에 필요 없는 대용량 증빙 사진 데이터 조회 제외
- 모바일에서 일시적인 `Load failed` 또는 서버 503·520 발생 시 조회를 자동 재시도
- 고객 개통일, 해피콜 대상일·담당자·매장, 처리 이력, 감사로그, 프리패스 로그 조회용 DB 색인 추가
- 해피콜 생성 규칙·배정 규칙·저장 방식은 변경하지 않음

## V29.51 QA 결과

- 자동 테스트 9건 통과
- 실제 운영 DB 기준 전체 해피콜 6,406건과 화면 집계 6,406건 일치
- 전체 해피콜·직원별 현황·통화 불가 고객·해피콜 생성·감사로그·오류보고 화면 진입 확인
- 프리패스 전체 현황·프리패스 로그 화면 진입 및 페이지 이동 표시 확인
- PC·390px 모바일 화면에서 오류 메시지 및 화면 중단 없음
- 운영 배포 전 최종 확인 단계

## V29.50 변경사항

- 지축 판매건의 판매 담당자가 퇴사 상태이면 가상 매장 대신 재직 중인 임지하 직원에게 직접 배정
- 지축 폐점 매장은 별도 승계매장 없이 지정 배정 규칙만 적용
- 퇴사 처리된 직원의 기존 PC·휴대폰 로그인 상태를 실행 시·화면 복귀 시·30초 주기로 재확인하여 자동 종료
- 브라우저 저장공간의 로그인 정보에서 직원 비밀번호 제거
- 동일 오류보고가 여러 탭에서 동시에 접수되는 상황 추가 차단
- 홈 화면의 내 해피콜·검수 숫자 조회 오류 수정
- 1280px PC 상단 메뉴 한 줄 유지 및 모바일 홀수 카드·프리패스 탭의 빈 공간 제거
- 악세사리 주문 입력칸의 명칭을 보조기기에서도 명확히 인식하도록 개선
- 신규 직원 등록 시 자동 기본 비밀번호 노출을 없애고 관리자가 초기 비밀번호를 직접 입력하도록 개선
