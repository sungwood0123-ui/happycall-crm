# V29.56 권한 기준선

V29.57 보안 강화는 아래 업무 권한을 바꾸지 않는 것을 전제로 한다. 이 문서는 화면 숨김뿐 아니라 데이터 조회·저장 정책과 회귀 테스트의 기준으로 사용한다.

## 공통

- 모든 재직자: 홈, 프리패스 기본 기능, 악세사리 주문, 알림 설정, 건의/문의, 사용방법, 비밀번호 변경, 로그아웃
- 퇴사 또는 리스트 제외: 로그인 및 모든 업무 접근 차단
- 비밀번호 변경이 필요한 재직자: 비밀번호 재설정 화면 외 업무 접근 차단

## 직원

- 내 해피콜 조회·처리
- 본인 프리패스 조회·신청·적립 요청
- 기존 악세사리 주문 업무
- 본인 건의/문의와 알림 설정

## 점장

- 직원 권한에 더해 현재 소속 매장의 해피콜 현황·리스트·직원별 현황
- 현재 소속 매장의 프리패스 점장 승인과 전체 현황
- D+93/D+183 대상의 기존 임시 배정 기능

## 검수자

- 내 해피콜
- 허용 매장에 대한 검수·전체 해피콜·전체 직원 현황
- 관리자 전용 직원·매장·RAW·생성·감사·오류 관리 기능은 접근 불가

## 관리자

- 검수, 전체 해피콜, 배정 현황, 전체 직원 현황
- 통화 불가 고객, RAW 업로드, 해피콜 생성
- 직원관리, 매장관리, 감사로그, 오류보고
- 현재 UI에서 관리자에게만 표시되는 가입번호 수정 및 당일 생성 해피콜 삭제
- 프리패스 관리자 권한 범위

## 최고관리자

- 관리자 공통 업무와 프리패스 최종 승인·관리자 조정·월 한도·반기 초기화
- 내 해피콜 탭은 표시하지 않음
- 최고관리자 여부는 이름이나 이메일이 아니라 DB의 `최고관리자` 권한만 기준으로 판정

## 변경 금지 기준

- 보안 강화 과정에서 위 메뉴가 새로 생기거나 사라지면 안 된다.
- 점장이 다른 매장 데이터, 직원이 다른 직원 데이터, 검수자가 허용되지 않은 매장 데이터를 보거나 수정할 수 없어야 한다.
- 화면에서 버튼을 숨기는 것과 DB에서 실제 접근을 차단하는 조건이 같아야 한다.
- 비밀번호·인증 토큰·임시 비밀번호를 감사로그, 오류보고, 브라우저 저장소에 남기지 않는다.
